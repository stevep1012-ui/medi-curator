# IMPROVE-2026-06-14 — 상업적 완성도 개선팀 사이클 1

> 산출: 레드팀(5축 병렬 감사, 37개 원시 발견) → 트리아지(33개 우선순위) → 개선 로드맵.
> 자율 정책(사용자 승인): **감사 + 로드맵 작성, 코드는 safe-auto 항목만 자동 적용.** needs-approval / 외부블로커는 결정 대기 큐로만 보고.

## 현재 상태 요약

medi-curator는 아키텍처 기반(서버 전용 LLM 프록시, uid 단위 rate limiting, App Check)은 견고하나, **상용 완성도 관점에서는 미완성이며 RELEASE HOLD가 정당화되는 상태**다. 두 개의 P0가 충돌하면서 제품이 동시에 "동작 불가"이면서 "안전하지 않은" 상태에 놓여 있다.

**코드 블로커 (자체 수정 가능)**

- **배포된 Firestore 규칙이 모든 동의 쓰기를 거부** — `firestore.rules:14`는 `acceptedAt == request.time`(서버 Timestamp)을 요구하지만 클라이언트(`consentService.ts:19`)는 `new Date().toISOString()`(문자열)을 쓴다. 문자열은 `request.time`과 결코 같을 수 없어, 규칙 배포 시 모든 동의 생성이 거부되고 `curate`가 `CONSENT_REQUIRED(403)`을 반환 → **핵심 큐레이션 플로우 100% 불능**.
- 동의 버전 client/server 분리 결합(`index.ts:131` 하드코딩 `'2026-06-01'`), DSR 엔드포인트 무방비(App Check/rate limit 부재 + 단일 토큰으로 Auth 계정 영구 삭제), ErrorBoundary 부재(렌더 에러 시 전체 화면 백스크린), 핵심 증상 분석 플로우 한국어 전용(4개 언어 광고에도 위기 블록까지 ko 고정).
- 릴리스 게이트(`gate:release-readiness`)가 `release-profile.json`만 검사하고 `Legal.tsx`/`index.html`을 스캔하지 않아 **위약(placeholder) PIPA 연락처·미검증 의료 주장이 살아있는데도 GO를 인증할 수 있음** (게이트 자체 신뢰 불가).
- 인플레된 커버리지(존재하지 않는 `pharmacyService` 소스를 검증하는 vacuous 테스트), 운영되지 않는 R-005 위기 회귀(shipped UI가 아닌 미배포 `SymptomInput` 검증).

**외부 블로커 (법무/임상/사업 결정 필요)**

- **InteractionCheck 탭의 개인화 약물 상호작용 판정**(danger/caution/safe) — RT-005가 출시 전 제거를 지시한 무면허 의료행위 노출이며, 앱 자체 면책 문구(`i18n.tsx:158/283`)와 정면 모순. 특히 9-rule 미매칭 조합에 "특이사항 없음/안전" 녹색 체크를 표시해 위험 조합에 대해 거짓 안전 보장을 제공.
- 미검증 "약사 감수" 주장(TrustStrip 전 화면 + `index.html` 메타), 비타민 페어링 효능/복용시점 처방성 카피, device-only 저장 약속 vs 서버 처리 고지 충돌(동의 유효성 약화 / PIPA·GDPR 정확성 리스크).
- 사업 모델 부재 — FAQ는 프리미엄을 약속하나 pricing/plan/billing/계정관리 surface가 전무. **HOLD가 풀려도 현재 상태로는 판매 불가.**

핵심 메시지: safe-auto 빠른 개선(ErrorBoundary, i18n 스레딩, 에러 현지화, CSP 축소, dead-affordance 정리, 가짜 데이터 제거)은 HOLD를 건드리지 않고 완성도를 즉시 끌어올린다. 그러나 진짜 출시 준비는 needs-approval 법무/의료/동의 항목 해소에 달려 있다.

## 1차 사이클 즉시 처리(safe-auto)

HOLD(의료/법무) surface를 건드리지 않고 완성도를 올리는 즉시 착수 항목:

- **루트 ErrorBoundary 추가** (`main.tsx`) — `<Home/>` 래핑, 현지화 fallback(사과 + 새로고침 + 119 안내). 백스크린 제거.
- **증상 분석 i18n 스레딩** (`SymptomAnalysis.tsx`) — 하드코딩 문자열 Dict 이관, `useI18n()` 언어를 `getCurationFromGemini`에 전달, lang 전달 테스트 추가.
- **위기 시 제출 차단** (`SymptomAnalysis.tsx`) — `instantEmergency==='mental'`에서 분석 버튼 비활성 + 청소년 1388 추가.
- **런타임 에러 현지화** (`geminiService.ts`) — 에러 문자열 Dict 키화, 활성 언어로 해석.
- **CSP 축소** (`firebase.json`) — `generativelanguage.googleapis.com` 제거 + 와일드카드를 실사용 호스트로 좁히기.
- **약국 finder 정리** (`PharmacyFinder.tsx`) — placeholder/title 현지화, Directions 딥링크 또는 제거, 카드별 "데모 데이터" 배지.
- **검색 이력 가짜 seed 제거** (`SearchHistory.tsx`) — 실데이터 없으면 `h.empty`, delete aria-label 현지화.
- **죽은 affordance 정리** (`Chrome.tsx`) — Support→`LegalModal` 연결, Preferences 제거(또는 숨김).
- **Welcome 토스트 순서 수정** (`page.tsx`) — `signIn` await + 성공 시에만 토스트.
- **로그인 모달 a11y** (`Chrome.tsx LoginGate`) — `role=dialog`/aria-modal/포커스 트랩/Esc.
- **SEO/PWA 자산** (`public/`, `index.html`) — robots/sitemap/manifest/OG·Twitter/canonical (메타 문구는 de-escalate 후 적용).
- **허구 테스트 삭제 + 큐레이션 테스트 추가** — `pharmacyService.test.ts` 제거, `SymptomAnalysis` 위기/에러 분기 테스트.
- **LLM JSON 파싱 견고화** (`index.ts`) — `responseMimeType='application/json'` 설정 + 추출 테스트.
- **큐레이션 캐시 경계** (`geminiService.ts`) — 소형 LRU + TTL.

> 주의: SEO 메타와 TrustStrip 문구는 법무 de-escalation 문구가 확정된 뒤 적용한다.

## 반복 사이클 운영

상용 완성도를 지속적으로 끌어올리기 위한 **red-team → 개선팀** 반복 루프. 모든 산출물은 `docs/reviews/`에 기존 네이밍 컨벤션(`{ROLE}-{YYYY-MM-DD}.md`)으로 적재한다.

**케이던스:** 1주 1사이클(코드 변동 활발 시), 또는 needs-approval 항목 외부 결정 도착 시 트리거.

**각 패스가 하는 일**

1. **Red-team 패스 (`REDTEAM-{날짜}.md` / 후속은 `REDTEAM-FOLLOWUP-{날짜}.md`)** — 직전 follow-up을 먼저 읽어 종결 항목 재보고 방지. 의료·법무 안전, 인증·남용, 프라이버시, 신뢰 일관성 4축으로 라이브 surface를 grep-검증(파일·라인 인용 필수).
2. **개선팀 패스 (`IMPROVE-{날짜}.md` — 본 로드맵 형식)** — red-team 발견을 트랙 A/B/C로 분류, 자율성·공수 부여. safe-auto는 당 사이클 구현 + 커밋 링크 기록. needs-approval/외부블로커는 결정 대기 큐로 명시.
3. **게이트 패스 (`RELEASE-BLOCKERS-{날짜}.md`)** — 남은 P0/P1을 출시 차단 목록으로 동기화. 게이트 자체 신뢰성(`gate:release-readiness`가 placeholder 토큰 스캔 여부)을 매 사이클 점검.

**추적 방법**

- 각 발견은 안정 ID를 유지하고 사이클 간 상태를 `open / in-progress / closed / external-blocked`로 갱신.
- 개선팀 문서 상단에 "직전 사이클 대비 변화"(신규/종결/회귀) 섹션 — 회귀 조기 포착.
- needs-approval/외부블로커 항목은 결정자(법무/임상/사업)와 요청 일자를 명시.
- 완성도 지표 추이 기록: 라이브 surface P0/P1 수, safe-auto 종결률, 실코드 커버리지, 게이트 스캔 통과 여부.

**루프 종결 조건(출시 GO):** 라이브 surface P0 0건 + 모든 외부블로커 closed + 게이트가 placeholder·App Check·동의 버전을 검증한 상태에서 GO.
